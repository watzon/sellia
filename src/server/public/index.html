Features

Everything You Need

Built for developers who need reliable, secure tunnels without the complexity.

◈

Subdomain Routing

Each tunnel gets its own subdomain. Request myapp and access your local server at myapp.your-domain.com.

⬡

Request Inspector

Built-in web UI to inspect all requests and responses in real-time. Debug webhooks with ease.

◇

Auto Reconnect

Lost connection? The client automatically reconnects with exponential backoff. No manual intervention needed.

△

Basic Auth

Protect your tunnels with built-in basic authentication. Keep your development previews private.

▽

Rate Limiting

Server-side rate limiting protects against abuse. Configure limits to match your needs.

◎

Binary Protocol

MessagePack-based protocol over WebSocket for efficient, low-latency communication.

Quick Start

Up and Running in Minutes

Get started with Sellia in three simple steps.

1

Install from Source

Clone the repository and build with Crystal. Requires Crystal >= 1.10.0.

2

Start Your Local Server

Run your development server on any port, e.g., localhost:3000.

3

Create a Tunnel

Run sellia http 3000 and share your public URL.

terminal
# Clone and build git clone https://github.com/watzon/sellia.git cd sellia shards build --release # Create a tunnel to your local server ./bin/sellia http 3000 # With custom subdomain ./bin/sellia http 3000 --subdomain myapp # With basic auth protection ./bin/sellia http 3000 --auth admin:secret

Ready to Get Started?

Sellia is free, open-source, and self-hostable. Run your own tunnel server or use the hosted service.
